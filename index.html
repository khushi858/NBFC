    Zoho Creator integration notes (for the developer wiring this up)

    This site is deliberately built with no backend of its own — every form hands off to a Zoho Creator public form instead of posting anywhere on this page. Three integration points, all following the same pattern:

    1. Where the config lives
    const ZOHO = {
  applyFormUrl:   "https://creatorapp.zoho.in/OWNER/APP_LINK_NAME/form-embed/APPLY_FORM_LINK_NAME",
  contactFormUrl: "https://creatorapp.zoho.in/OWNER/APP_LINK_NAME/form-embed/CONTACT_FORM_LINK_NAME",
  portalUrl:      "https://creatorapp.zoho.in/OWNER/APP_LINK_NAME/PORTAL_PAGE_LINK_NAME"
};

    Replace OWNER, APP_LINK_NAME and the form link names with your actual values from Zoho Creator → your app → Forms → the specific form → Embed. Search for const ZOHO = near the bottom of this file's script tag and edit it there.

    2. Pre-filling the embedded form

    Zoho Creator's public forms accept field values via URL query parameters matching each field's link name, e.g. ?Full_Name=Khushi&Loan_Amount=500000. The script below builds that query string from the wizard's fields — you only need to rename the keys in fieldMap to match your actual Creator field link names (check them under each field's Advanced settings in Creator's form builder).

    3. Why an iframe embed instead of a direct API call

    Posting straight to Creator's REST API from browser JavaScript would mean exposing an API token to anyone who views this page's source — a real security problem. The embed approach keeps the token server-side (Zoho hosts it), and the form still submits directly into your Creator report, ready for your existing LOS workflow to pick up.

    4. Customer portal / login

    OTP-based login needs session handling, which belongs in a Creator custom page or a small Deluge-based API — not in static HTML. The "Log in" button here just opens ZOHO.portalUrl in a new tab with the mobile number appended, so your Creator app can pick up from there.
